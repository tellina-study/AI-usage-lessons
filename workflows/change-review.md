# Change Review Process

## Create-Roast-Revise

All non-trivial changes follow this cycle:

1. **Create** — draft the change (document edit, new lecture, diagram update)
2. **Roast** — critically review: accuracy, completeness, consistency with existing materials
3. **Revise** — incorporate feedback, fix issues found during roast

## Review Checklist

- [ ] Change is linked to a GitHub Issue
- [ ] Ontology relations are updated if entities changed
- [ ] No orphaned references (documents citing removed sections)
- [ ] Diagrams still match the content they illustrate
- [ ] Manifests updated if new artifacts were created
