# План задачи: ревью и улучшение Лекции 1

**Issue:** #47
**Ветка:** `issue-47-lecture-1-review`
**Источник:** https://docs.google.com/document/d/1lu_xcn4IA2sgOIsrFDZ1kdawkCXGCcnyqTIjA4zY3VM/edit (план слайдов v3.1)
**Цель:** сделать логичную, цепляющую, пригодную к проведению лекцию, соответствующую лучшим практикам публичных лекций.

## Фазы с явными гейтами

### Фаза A — Загрузка и фиксация исходника
- Экспортировать Google Doc в markdown.
- Сохранить в `catalog/exports/docs/lec-01-slides-plan-v3.1.md`.
- **Gate A:** файл в репо, коммит сделан, пользователь подтвердил, что это корректная версия.

### Фаза B — Прожарка
- Острая критика плана по осям: драматургия, когнитивная нагрузка, релевантность, точность 2026, слайд-дизайн, интерактив, академ-антипаттерны, локальные больные места.
- Минимум 10 пунктов с цитатами из исходника + приоритизация (P0/P1/P2).
- Артефакт: `notes/lecture-1-review/roast.md`.
- **Gate B:** пользователь прочитал прожарку, отметил какие пункты принимает/отклоняет, добавил свои.

### Фаза C — Сбор замечаний пользователя
- Свести пользовательские правки и приоритеты в `notes/lecture-1-review/user-notes.md`.
- **Gate C:** согласованный список задач для синтеза улучшенной версии.

### Фаза D — Синтез улучшенной версии
- Делегировать субагенту (`doc-editor` или `deck-editor`) создание улучшенного плана слайдов.
- Артефакт: `notes/lecture-1-review/improved-plan.md` + (опционально) обновление Google Doc через `workspace-mcp` в отдельной версии.
- **Gate D:** пользователь прочитал улучшенную версию, одобрил для PR.

### Фаза E — Ревью и PR
- Открыть PR из `issue-47-lecture-1-review` в `main`.
- Дождаться явного одобрения от пользователя.
- Мердж выполняет пользователь (или с явного разрешения).
- **Gate E:** PR смерджен, issue закрыт.

## Правила

- Все коммиты с пометкой `#47`.
- Не пушить в main напрямую.
- Не мерджить свой PR.
- Не переходить между фазами без явного "go" от пользователя.
